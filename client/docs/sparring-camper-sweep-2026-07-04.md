# 陪练基建与 camper 局参数重扫（2026-07-04，V3.19）

> 前情：首轮镜像扫描（arena-sweep-2026-07-04.md）发现反蹲点/设卡类参数在
> 镜像局全部不绑定。本轮补上风格化陪练，把这些参数逼进测试范围。

## 一、陪练（client/sparring.py）

- **CamperBot**：复刻 replay36 §4 的 2614 六步套路——直奔武关、清障铺路、
  设满防卡、蹲卡后农任务、卡被拆原地补、尾段路过潼关回手第二张卡再交付。
- **RusherBot**：零贪婪竞速——最短路直冲，顺手骑马/捡零绕路冰，蹲 RUSH 验核。
- 接入：`arena.py --opp camper|rusher`、`sweep.py --opp camper`。
- 陪练是确定性脚本，用于回归与参数绑定，不追求强度。

## 三、camper 局参数重扫（每条件 48 局）

基线 win 0.88 / margin +108 / 未交付 6（48 局双座位）：

| 参数 | 结果 | 解读 |
|---|---|---|
| **`CAMPER_GRACE` 8→5** | **win 1.00 / +192 / 未交付 0** | 本轮唯一的免费午餐，已采纳（§四） |
| `RACE_FRAME_MULT` 1.25 | 0.75 / +25 / 12 死 | 竞速定价对蹲点流保命；镜像局它是中性的 |
| `RACE_FRAME_MULT` 2.3 | 0.92 / +121 / 4 死 | 对 camper 更好——但镜像局 0.25 惨案 |
| `FUNNEL_GUARD_PRIOR` 0.91 | 0.92 / 4 死（0.49 → 12 死） | 对 camper 越悲观越好——与镜像完全相反 |
| `SWITCH_MARGIN` 1.1 | 0.92 / 4 死（1.3 → 12 死） | 同上，方向与镜像相反 |
| 其余（设卡三参数/TRAP_AVOID/LOITER 等） | 与基线逐位相同 | 对确定性 bot 仍不绑定 |

**关键观察**：`RACE_FRAME_MULT`↑ / `SWITCH_MARGIN`↓ / `FUNNEL_GUARD_PRIOR`↑
三组的结果数字完全相同（0.92/+121/4 与 0.75/+25/12 两簇）——它们拨动的是
**同一个漏斗前分叉**（"多做一个任务再动身 vs 提前动身"），且最优方向与镜像局
相反。参数没有全局最优值，**最优值依赖对手风格** → 对手画像分类器
（竞速型/蹲点型/农任务型 → 切换参数档）从"建议"升级为"有数据立项"。

## 四、V3.19 采纳的改动

1. **`CAMPER_GRACE` 8→5**：语料侧（V3.17 记录的临别卡对手全部 1~4 帧内
   离开：2614 r314→318 / r322→323，2839 r309→310 / r450→451）与竞技场侧
   （0 死、+84 分/局）双重证据；镜像局该参数不绑定（±30% margin 恰 0）。
   3 帧的级联效应：早 3 帧出武关 → 赶在 camper 到潼关起卡前上边 →
   整段 45 帧汇聚等待消失，这是 3 帧翻活 20 帧缺口的机制。
2. **坐地户免宽限**：起卡前已驻扎 ≥`CAMPER_ESTABLISHED`(20) 帧的对手不是
   临别过客，新卡直接强通。对"到点即起卡"的 bot 不触发（它起卡前驻扎仅
   ~6 帧），防的是 2614 式"农一会儿再起卡"的真坐地户。selftest 12/12b 对。

复核战绩：**vs CamperBot 24/24（未交付清零）、vs RusherBot 12/12、
镜像 6/1/5 均衡、selftest 194 项全绿。**
